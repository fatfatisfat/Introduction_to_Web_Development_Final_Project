6-1 常見的網路服務
核心概念：介紹網際網路基礎服務，如全球資訊網（WWW）、電子郵件（E-mail）、檔案傳輸（FTP）、雲端儲存（Google Drive, OneDrive）等。
應用場景：日常工作溝通、遠端檔案備份與同步協作。

核心概念：引導如何精準獲取網路資訊。包含關鍵字邏輯、布林邏輯（AND, OR, NOT）、雙引號精準搜尋及進階搜尋篩選器（按時間、檔案類型、地區）。
應用場景：學術報告資料搜集、事實查核。

6-3 數位學習
核心概念：打破時空限制的自主學習模式。介紹大規模開放式線上課程（MOOCs，如 Coursera, edX）與國內外數位學習平台（如 均一教育平台、磨課師）。
應用場景：跨領域技能進修、課後進度複習。

6-4 網路影音資源應用
核心概念：串流媒體（Streaming Media）的運作與應用。涵蓋影音平台（YouTube, Netflix）與音樂串流（Spotify, KKBOX），並探討版權與合理使用（Fair Use）概念。
應用場景：娛樂視聽、影音素材合法剪輯。

6-5 自媒體與社群媒體
核心概念：個人傳播時代的興起。分析自媒體（We-Media）的定義，以及傳統社群（Facebook, Instagram）與短影音社群（TikTok, Instagram Reels）的傳播特性。
應用場景：個人品牌經營、社群小編行銷。

6-6 好用的網路資源
核心概念：日常實用工具的整合介紹，如線上翻譯、公共資訊平台（政府開放資料）、免費開源素材庫（Pixabay, Unsplash）。

筆記整理

第一部分：網路類型、拓樸與硬體設備


1. 網路類型（依地理範圍區分）

區域網路 (LAN)： 範圍 \(10 \text{ 公里}\) 內。組織、企業內部專用。

都會網路 (MAN)： 範圍 \(50 \text{ 公里}\) 內。由數個 LAN 相連而成，例如：有線電視網路。

廣域網路 (WAN)： 範圍 \(50 \text{ 公里}\) 以上。橫跨城市與國家，例如：臺灣學術網路 (TANet)（於民國 79 年 / 1990 年建立，為台灣科技教育打下深厚物理基礎）。

2. 區域網路拓樸架構

匯流排狀 (Bus Topology)： 所有裝置接在同一條主線路

星狀 (Star Topology)： 最常見的架構，透過中央節點交換訊息。

環狀 (Ring Topology)： 裝置連成圓環，單向傳輸。

3. 網路硬體設備核心功能

網路介面卡 (NIC)： 負責平行與串列資料轉換。每張卡有全球唯一的 MAC 位址（\(6 \text{ Bytes}\) / \(48 \text{ bits}\)，十六進位）。（Windows CMD 查詢指令：ipconfig /all）

中繼器 (Repeater)： 強化衰減的訊號。

集線器 (Hub)： 多埠中繼器，所有連接埠共享頻寬，效能較低。

交換器 (Switch)： 依 MAC 位址傳送資料，每個連接埠擁有專屬頻寬，效率高（實作於資料連結層）。

IP 分享器： 讓多個設備共用一個合法 IP 位址（常結合 NAT 技術）。

路由器 (Router)： 提供資料傳輸的路徑選擇 (Routing)（實作於網路層）。

閘道器 (Gateway)： 連接兩個通訊協定完全不同的網路。

第二部分：網路計量與無線通訊技術


1. 網路速度與檔案大小換算

核心關鍵： 區分 Bit (位元) 與 Byte (位元組)。公式\(1 \text{ Byte} = 8 \text{ bits}\)

Mbps： 傳輸速度單位（Megabits per second），指每秒傳輸多少「位元」。

MB： 檔案大小單位（Megabytes），指資料儲存的總量。

實戰計算範例：
\(5\text{MB}\) 歌曲在\(10 \text{Mbps}\) 網速下\(5 \times 8 = 40\text{ Mb} \rightarrow 40 / 10 =\)\(4 \text{ 秒}\)

\(1\text{GB}\) 影片在\( 100\text{Mbps}\) 網速下\(1024 \times 8 = 8192\text{ Mb} \rightarrow 8192 / 100 \approx\)\(82 \text{ 秒}\)

2. 無線網路四層級分類

無線廣域網路 (WWAN)： 跨城市、跨國。核心為蜂巢式網路 (Cellular Network)，基地台訊號覆蓋呈「六邊形」以確保移動不斷線。演進：3G (HSPA)\(\rightarrow\) 4G (LTE/LTE-A) \(\rightarrow\) 5G (NR)。組成包含：移動站、基地台子系統、網路子系統。

無線都會網路 (WMAN)： 整個城鎮。標準為 IEEE 802.16 (WiMAX)。

無線區域網路 (WLAN)： 辦公室、校園或住家。核心為 Wi-Fi (IEEE 802.11 系列)。

Wi-Fi Direct： 點對點連接技術，不需透過路由器即可互傳，速度高達\(250\text{Mbps}\)。

無線個人網路 (WPAN)： 個人週邊數公尺內。

藍牙 (Bluetooth)： IEEE 802.15.1。低功耗、傳輸音訊或短資料。

超寬頻 (UWB)： 精準定位、高安全性、高速傳輸。

ZigBee： IEEE 802.15.4。低成本、支援大量節點（智慧家居常用）。

3. NFC 與 RFID 技術辨析

RFID (無線射頻辨識)： 無方向性限制、可遠距離批次讀取。其電子標籤 (Tag) 分三種：
主動式 (Active)： 自備電池，主動發射，範圍最廣。

半被動式 (Semi-Passive)： 有電池，但需讀取器觸發才工作。

被動式 (Passive)： 無電池，靠讀取器的電磁波感應供電（如超商標籤）。

NFC (近場通訊)： 從 RFID 演進的特化技術。距離極短（\(< 10\text{cm}\)），安全性高。

三種操作模式： 卡片模擬（手機當悠遊卡/信用卡）、讀寫模式（讀取標籤）、點對點模式（兩台設備交換資料）。

第三部分：OSI 模型與網路識別 (IP / MAC / DNS)


1. OSI 七層參考模型 (ISO 制定)

7. 應用層 使用者操作介面、提供網路服務。 HTTP, FTP, LINE, DNS

6. 表示層 資料格式轉換、加密、解密、壓縮。 JPEG, MP4, SSL/TLS

5. 會議層 建立、管理、維護與拆除連線對話。 RPC, SQL

4. 傳輸層 端到端傳輸、流量控制、錯誤恢復。 TCP (可靠), UDP (快速)

3. 網路層 邏輯定址、決定傳輸路徑（路由）。 IP, 路由器 (Router)

2. 資料連結層 實體定址、點對點傳輸、錯誤偵測。 MAC 位址, 交換器 (Switch)

1. 實體層 傳輸原始位元 (Bit) 流，定義硬體規格。 網線, 光纖, 集線器 (Hub)

2. IP 位址管理架構

核心對比： MAC 位址是身分證字號（\(48 \text{ bits}\)，出廠固化全球唯一）；IP 位址是通訊住址（隨環境改變）。

IPv4 結構與五大分級： 長度 \(32\text{ bits}\)。由網路位址 (Network ID) 與主機位址 (Host ID) 組成。系統將 IP 與子網路遮罩進行 AND 邏輯運算 即可得出網路位址。

Class A： 1 ~ 126（遮罩 255.0.0.0）。適用國家級、超大型組織。

Class B： 128 ~ 191（遮罩 255.255.0.0）。適用中大型企業。

Class C： 192 ~ 223（遮罩 255.255.255.0）。適用小型辦公室、家庭。

Class D： 224 ~ 239。用於群播 (Multicast)（如視訊會議、直播）。

Class E： 240 ~ 255。實驗、研發保留。

特別備註： 127.0.0.1 (Loopback 環回位址) 用於與「自己電腦」通訊測試，不暴露於公網。

IPv6 規範： 長度 \(128\text{ bits}\) 十六進位（8組，冒號隔開）。

縮寫規則： 可省略前導零；連續全零群組可用雙冒號 :: 取代（單一位址限用一次）。

NAT (網路位址轉換)： 將多個「私有 IP」（如 192.168.x.x）轉換為「公有 IP」以存取外網。核心優勢：節省 IPv4 消耗、提供初步安全屏蔽、簡化內部網管。

3. 網域名稱系統 (DNS) 與 URL 架構

DNS： 採樹狀管理階層結構，將人類易記的域名轉為電腦識別的 IP（如：www.ntu.edu.tw \(\rightarrow\) 主機名 + 機構名 + 類別名 + 國家碼）。

URL 標準格式： 通訊協定://主機網址[:連接埠]/路徑/檔名

預設連接埠： http 為 80，https 為 443。參數以 ? 引導傳遞。

第四部分：寬頻上網技術與連線設定


1. 三大寬頻上網方式對比

ADSL (非對稱數位用戶線路)： 利用傳統電話線（銅線）傳輸。劃分高頻段傳數據，下載與上傳速率不對等。受傳輸距離物理限制嚴重，逐漸被光纖取代。

Cable Modem (纜線數據機)： 利用有線電視同軸電纜。需將類比與數位訊號轉換。同一社區用戶共用頻寬（尖峰時段會略微降速）。

光纖上網 (Fiber Optics)： 利用雷射光信號的全反射原理在玻璃/塑膠纖維中傳輸。高頻寬、低延遲、高穩定、抗電磁干擾。

FTTH (光纖到家)： 直接拉入家中，最穩定高速，適合 4K 影音、線上遊戲與隨選視訊 (VOD)。

FTTB (光纖到大樓)： 連至大樓電信室，再透過銅線或乙太網線分接到各戶。

FTTC / FTTCab (光纖到街角/電信箱)： 僅到達道路旁電信箱，最後一段採其他介質。

2. 區域網路拓樸架構

浮動 IP (Dynamic IP)： 連線時隨機指派，斷線回收。透過 DHCP (動態主機設定協定) 自動分派，省資源，適合一般家庭。

固定 IP (Static IP)： 每次連線都固定不變。適合架設網站、郵件伺服器、架設遊戲主機等需要遠端連線的進階用戶。

第五部分：WWW、HTML5 前端開發與 Google Sites


1. 全球資訊網 (WWW) 與網頁基礎

WWW 背景： 1989 年由 CERN（歐洲核子研究組織）研發，運用超文本技術 (Hypertext) 整合多種協定。

超文件 vs 超媒體： 透過「超連結」互相串接的文件稱為超文件；在電腦上以多媒體形式呈現的稱為超媒體。主要使用 HTML 製作。

HTML5 特色： 2014 年完成標準化。文件第一行必須是文件類型定義（DTD）宣告：，藉此告訴瀏覽器採用 HTML5 標準解析。

第六部分：網路世代演進與物聯網 (IoT) 大數據架構


1. 網路世代演進史 (Web 1.0 到 Web 3.0)

Web 1.0 (唯讀型 Read)： 靜態網頁，使用者被動接收資訊。數據由網站主所有（如早期入口網站）。

Web 2.0 (讀寫型/互動型 Read+Write)： 社群媒體興起（FB、YouTube），使用者產製內容。數據權力集中在平台巨頭（中心化）。

Web 3.0 (擁有型 Read+Write+Own)： 基於區塊鏈實現去中心化，數據與數位資產由使用者本人所有。

區塊鏈 (Blockchain)： 底層信任協議。利用分散式帳本確保資料不可竄改且公開透明，去除第三方仲介。

NFT (非同質化代幣)： 數位資產的唯一性與稀缺性證明，確認鏈上原創所有權，解決易複製問題。

DApp (去中心化應用程式)： 運行在區塊鏈上，無單一伺服器，消除單點故障 (Single Point of Failure) 風謝，防止內容審查或隨意改規。

2. 物聯網（IoT）架構演進：三層 -> 現代五層

物聯網架構從早期的歐洲電信標準協會 (ETSI) 三層架構，演進為結合大數據運算的現代五層架構：

[ 商務層 (Business Layer) ] --> 企業決策與商業智慧 (BI)
    ↓
[ 應用層 (Application Layer) ] --> 使用者介面與智慧行業應用
    ↓
[ 處理層 (Processing Layer) ] --> 海量數據分析、清洗與儲存 (Data Mining/DW)
    ↓
[ 傳輸層 (Transport Layer) ] --> 數據安全路由與網絡傳遞
     ↓
[ 感知層 (Perception Layer) ] --> 環境感測 (Sensor) 與身分辨識 (RFID/NFC)

3. 物聯網與大數據核心技術剖析

A. 感知層技術（物聯網的根基）

    感測技術： 賦予物體感受環境變化或移動的能力。

    感測器 (Sensor)： 探測外界反應並轉化為可量化訊號。

    3D 感測器： 結合影像感測、紅外線與演算法，精準捕捉特定影像，不受混亂背景或距離干擾。

    辨識技術： 以 RFID 為核心，自動獲取裝置或物品的身分與當前狀態。

B. 應用層與商務層技術（大數據智慧決策）

    此處的核心在於如何將感知層收回來的海量數據轉化為黃金，主要包含以下四大技術：

    資料倉儲 (Data Warehouse, DW)： 將多個相異的資料來源經過篩選分類後，整合儲存於大型資料庫中，支援中長期決策。

    線上分析處理 (OLAP)： 用於大型資料庫的資料分析、統計與多維度計算。其中「維 (Dimension)」是指人類觀察客觀世界的角度（如時間、地區、產品）。

    資料探勘 (Data Mining)： 運用統計、機器學習等技術對海量資料進行深層分析，自動擷取資料庫中隱含、有用且具關聯性的資訊或法則。

    商業智慧 (Business Intelligence, BI)： 整合 DW、OLAP、Data Mining，將企業內部數據轉化為直觀的分析結果與行動指引，輔助高層進行最終決策。

核心技術


1. 視覺捕捉與動態追蹤技術（Motion Capture）

這是讓虛擬模型「動起來」的關鍵，依據主流呈現方式分為 2D 與 3D：

1. 2D 臉部與上身追蹤：

技術核心： 利用一般網路攝影機（Webcam）或智慧型手機（如 iPhone 的 TrueDepth 鏡頭），透過電腦視覺演算法（如 Google MediaPipe、Apple ARKit）捕捉真人的面部表情、眨眼、轉頭等細微動作。

軟體： 最常使用 Live2D Cubism 將 2D 插畫進行分層與骨架綁定，並透過 VTube Studio、PrprLive 等軟體進行即時驅動。

2. 3D 全身動作捕捉：

光學式動捕（Optical MoCap）： 在專門攝影棚內，透過多顆高階紅外線攝影機捕捉中之人身上反光點的軌跡（如 Vicon、OptiTrack）。精準度極高，多用於大型 3D 演唱會。

慣性動捕（Inertial MoCap）： 讓中之人穿戴內建陀螺儀與加速度計的感測套裝（如 Rococo、Xsens）。優點是不受場地限制，適合居家 3D 直播。

2. 即時渲染與即時互動技術

遊戲引擎應用： 3D VTuber 的場景與模型多在 Unity 或 Unreal Engine 中進行即時渲染。這讓虛擬主播能與 3D 物件、特效（如觀眾投擲的虛擬禮物）產生即時物理碰撞。

語音處理技術： 包含即時變聲器（Voice Changer）或音訊修飾，用於保護中之人隱私或符合角色人設。
